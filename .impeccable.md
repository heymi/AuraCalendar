# AuraCalendar — Design Context

## Design Context

### Users
个人用户，在日常生活与工作中管理任务和计划。使用场景多样：早晨规划一天、快速记录想法、回顾跨天项目。用户希望打开 app 后立刻感到秩序与掌控感，而不是被信息轰炸。核心任务：快速创建任务、一眼看到本月安排、轻松切换任务状态。

### Brand Personality
**三个词**：克制、呼吸、可信

- **克制**：不过度设计，每一个元素都有存在的理由
- **呼吸**：留白是设计的一部分，界面让人心跳稳定
- **可信**：交互反馈准确，动画有物理感，从不让用户感到困惑

### Aesthetic Direction
- **参考**：iOS 原生日历、Apple Reminders、Fantastical — 系统级质感、毛玻璃层次、SF 风格排版
- **反参考**：避免过于鲜艳的渐变背景、避免信息密度像看板软件（Linear/Jira）那样高、避免像游戏化工具一样的视觉奖励感
- **主题**：浅色 + 深色双模式，跟随系统自动切换
- **动画节奏**：Spring 物理，弹而不夸张，150–300ms 为主，感觉像真实物体在移动

### Design Principles

1. **空气感优先** — 宁可多留白，也不堆砌信息。每个 Day Block 要有呼吸空间，绝不拥挤。

2. **iOS 系统级质感** — 毛玻璃弹窗（`backdrop-blur` + 半透明背景）、语义化颜色（#007aff / #0a84ff 双模式蓝、#34c759 绿、#ff9f0a 橙）、细描边而非重阴影。

3. **中性色主导，强调色点缀** — 背景和卡片保持中性灰白/深灰，只在关键交互（今日高亮、状态圆点、FAB）上使用强调色。深色模式用 Apple 深灰系（#1c1c1e / #2c2c2e / #3a3a3c），而非纯黑。

4. **有感情的微动效** — 每一次交互都有响应，但从不打断思路。状态切换用颜色过渡而非位移，弹窗入场用 spring scale + fade，列表项用 stagger。

5. **细节即诚意** — 精确的行高、letter-spacing、图标对齐。圆角要一致（按钮 rounded-xl，卡片 rounded-2xl，弹窗 rounded-3xl）。阴影只用于需要层次的地方，不滥用。
